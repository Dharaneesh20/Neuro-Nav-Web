# NeuroNav Developer Contributions Guidelines

## Welcome to NeuroNav!

We're excited to have you contribute to NeuroNav, an autism support companion application.

## Code of Conduct

- Be respectful and inclusive
- Consider the diverse backgrounds of contributors
- Focus on what's best for the community
- Support and encourage collaboration

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Follow our development guidelines
5. Commit your changes with clear messages
6. Push to your fork
7. Submit a Pull Request

## Development Guidelines

### Code Style

- Use 2-space indentation
- Use meaningful variable names
- Write clear comments for complex logic
- Follow the existing code patterns

### Commit Messages

Use clear, descriptive commit messages:

```
Feat: Add calm score monitoring feature
Fix: Resolve authentication token expiry issue
Docs: Update API documentation
Style: Format code to match standards
Test: Add unit tests for auth module
```

### Branch Naming

- `feature/feature-name` - New features
- `fix/bug-description` - Bug fixes
- `docs/what-document` - Documentation
- `test/test-description` - Tests
- `refactor/what-refactored` - Refactoring

### Pull Requests

1. Provide a clear title and description
2. Reference related issues
3. Include screenshots/videos for UI changes
4. Ensure all tests pass
5. Request review from team members

## Testing Requirements

- Write tests for new features
- Ensure existing tests pass
- Aim for >80% code coverage
- Test edge cases

## Documentation

- Update README for major changes
- Document new API endpoints
- Add inline comments for complex logic
- Update CHANGELOG

## Performance Considerations

- Minimize API calls
- Optimize database queries
- Use pagination for large datasets
- Cache data appropriately

## Security

- Never commit secrets or API keys
- Validate user input
- Use parameterized queries
- Follow OWASP guidelines
- Report security issues privately

## Accessibility

- Use semantic HTML
- Ensure keyboard navigation
- Add alt text for images
- Test with screen readers
- Use Bootstrap icons (no emojis)

## Questions?

- Open a GitHub discussion
- Check existing issues
- Review documentation
- Contact the team

## Attribution

All contributors will be acknowledged in the project documentation.
